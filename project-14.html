Overview

I built the motion design guideline system for Vervemobile, defining how the app's interface should move — from micro-interactions to full-screen transitions — so animation stays consistent no matter who's building on top of it.

I set out core principles — timing, easing, hierarchy of motion — then broke them into reusable patterns for the moments that matter most: loading states, card transitions, button feedback, navigation. The result is a reference any designer or engineer on the team can pick up and apply, so the product's feel never has to be reinvented from scratch.
